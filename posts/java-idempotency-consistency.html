交易清算链路里的幂等与一致性设计

发布于 2026-01-30 · 阅读约 10 分钟

交易清算场景天然要求“不能重复扣款、不能漏账、可追溯”。本文给出一套 Java 后端可执行的设计路径。

1. 幂等键设计

业务幂等键建议包含 business_id + account_id + phase，并配唯一索引，确保重复请求只会命中同一记录。

2. 事务与消息

  • 本地事务写业务表 + Outbox 表。
  • 异步投递 MQ，消费端再次做幂等校验。
  • 失败重试使用指数退避，避免抖动放大。

3. 审计与回放

所有关键状态迁移记录操作人、来源系统和 trace_id。遇到争议时可以按 trace 全链路回放，快速定位责任点。

4. 线上治理

发布前做“重复请求压测 + 乱序消息压测 + 人工补单演练”，把风险前置。
