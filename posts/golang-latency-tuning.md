延迟治理不是单点优化，而是从入口到存储的系统工程。这里分享我在交易 API 场景下的一套排查与优化路径。

## 1. 快速定位瓶颈

- 先看 P95/P99 分布，再看单链路火焰图。
- 区分 CPU 饱和、I/O 等待、锁竞争三类问题。
- 把 `queue_wait` 和 `handler_cost` 分开统计。

## 2. 常见优化点

- 连接池：统一上限，避免突发下游雪崩。
- 对象复用：在热点路径使用池化降低 GC 压力。
- 锁优化：拆大锁、缩临界区、优先无锁数据结构。

## 3. 稳态策略

当服务进入高峰期，优先保证核心交易接口，非核心接口降级或限速。把“可用”放在“全部功能完整”之前。

## 4. 线上经验

每次优化后必须回归压测并记录性能基线，建立“版本 - 指标”对照，避免后续迭代性能回退。
